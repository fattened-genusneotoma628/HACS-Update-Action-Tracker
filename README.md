## What this tool does

HACS-Update-Action-Tracker upgrades your standard Home Assistant update process. Many software updates include hidden features or new settings that require manual work to activate. This tool tracks those updates and creates specific tasks for you. It ensures you use every new feature provided by your custom components and integrations. You no longer need to guess if an update requires extra steps. The tracker lists exactly what you need to do to complete the process.

## ⚙️ Installation steps

Follow these steps to add the tracker to your Home Assistant instance.

1. Locate the downloaded file on your computer.
2. Right-click the file and select "Extract All" to reveal the contents.
3. Open your Home Assistant file structure using your preferred method, such as the File Editor or Samba Share.
4. Locate the folder named `custom_components` inside your Home Assistant configuration directory. If this folder does not exist, create it.
5. Copy the contents of the tracker folder you extracted earlier into the `custom_components` folder.
6. Restart your Home Assistant instance. This step loads the new integration. 
7. Open your dashboard and navigate to the "Integrations" page.
8. Click "Add Integration" and search for "HACS-Update-Action-Tracker" to finish the setup.

## 📋 Using the tracker

Once installed, the tracker runs in the background. It monitors your updates automatically. When a component releases an update, the tracker checks the release notes for specific instructions. 

It generates a task list on your dashboard. You will see these tasks under your primary HACS view. Each task describes a specific action, such as updating a configuration file or enabling a new setting. 

Review the list after you permit an update. Click on each item to mark it as complete. This helps you keep your system organized and removes clutter from your dashboard.

## 🔍 Troubleshooting common issues

If you encounter problems during the installation or usage, try these steps:

- Check your logs: If the integration fails to load, go to the "System" menu in Home Assistant and select "Logs." Search for mentions of the tracker to identify errors.
- Permissions: Ensure your user account has enough permission to write files into the `custom_components` folder.
- Network access: If the tracker cannot find new updates, check your router settings. Ensure the component has permission to reach the internet to fetch data from GitHub.
- Browser cache: Sometimes the dashboard does not show new tasks immediately. Refresh your browser page or clear your cache to force a reload of the layout.
- Version mismatch: Ensure both Home Assistant and HACS are updated to their latest stable releases. Using outdated core software often causes integration conflicts.

## Customizing your view

You can change how the tracker displays your tasks. Open the "Dashboard" view and look for the configuration card generated by the tracker. You can drag this card to any page. 

Use the settings menu on the card to filter tasks. You can hide completed tasks or group them by integration. If you have many custom components, use these filters to keep your workspace clean. You can also prioritize urgent tasks by pinning them to the top of the list.

## Keeping the tracker updated

The tracker itself receives updates from time to time. You can manage these updates through HACS just like any other custom component. When an update appears, the tracker adds a task to your list. Follow the same procedure you use for other components. 

If a major update occurs, the tracker might send you a notification. Keep an eye on your Home Assistant notification center for these alerts. This ensures you always run the latest version with the most accurate tracking capability.

## Frequently asked questions

Does this tool change my system settings?
No. The tracker only reads data and suggests tasks. It does not modify your files or configuration unless you choose to follow the task instructions manually.

Can I use this for standard Home Assistant updates?
This tool focuses on custom components and integrations managed by HACS. It does not monitor the core Home Assistant software updates. 

Is this safe for my system?
The tracker runs within the secure environment of Home Assistant. It follows the same security standards as other custom integrations. 

Who maintains this repository?
The community manages this integration to ensure compatibility with recent Home Assistant versions. You can view the full history and contributors on the GitHub repository page at any time.

How do I remove the tracker if I no longer need it?
Go to the "Integrations" page in Home Assistant. Select the HACS-Update-Action-Tracker and choose "Delete." Remove the files from your `custom_components` folder to complete the cleanup. Restart your system to refresh the interface.
